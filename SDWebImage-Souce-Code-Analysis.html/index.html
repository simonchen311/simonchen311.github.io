SDWebImage源码阅读

开发
  • Source Code
  • iOS

SDWebImage这套开源库可以说是开发必备品了，于是写这篇文章对其内部主要源码的实现整体理顺一下。

UIImageView入口

我们在使用SDWebImage时，经常是利用它提供的接口直接在UIImageView或者UIButton上设置URL来下载图片并且显示出来

以UIImageView+WebCache为例，其内部实际上是一个深层次的调用

UIView入口

继续查看，可以发现sd_internalSetImageWithURL:placeholderImage:options:operationKey:setImageBlock:progress:completed:是UIView分类提供的方法，在这个方法里面，主要做了这么些事情：

  1. 通过私有方法sd_cancelImageLoadOperationWithKey:取消当前UIView上正在下载的图片
  2. 设置占位图片
  3. 通过SDWebImageManager来下载对应的url图片，并且在图片下载完成后在主线程上设置图片到当前的UIView中显示，并且执行对应的completedBlock

UIView对操作的管理

UIView的这个分类时专门用来管理当前View中的下载操作的

其内部维护了一个NSMutableDictionary：SDOperationsDictionary用来管理当前View上的操作，操作是一个遵守SDWebImageOperation协议的对象，每个操作以当前的类名为key，存放在字典中。

从上面的源码可以看到，每一个View在新下载一张图片的时候都会从字典中取出原本正在下载的操作，取消后并移除之。

SDWebImageManager

SDWebImageManager是一个单例类，它用来做了两件事情：

  • 利用SDImageCache来管理图片缓存
  • 利用SDImageDownloader来下载图片

SDWebImageManager处理代码有点长，但理解起来并不难，它的处理流程主要如下：

  1. 创建一个SDWebImageCombinedOperation操作对象，该对象遵守了SDWebImageOperation协议。
  2. 判断当前下载的url是否之前下载失败过，如果是一个之前下载失败的url并且在选项中没有指定对失败的url重新下载，那么他会直接返回，执行completionBlock，否则继续执行下面的代码
  3. 下载之前首先会利用SDImageCache去查找图片是否有缓存，如果有缓存则直接显示图片，没有则继续进行下载操作
  4. 利用SDImageDownloder来下载对应的图片，下载完成后通过回调来显示。

SDWebImageCombinedOperation

SDWebImageCombinedOperation是SDWebImageManager返回给UIView的操作对象，其内部封装了一个cancleBlock用来取消本次的下载任务。

在调用SDWebImageCombinedOperation的cancle方法时，实际上其内部就是调用了这个cancleBlock用来取消下载操作的。

SDImageCache

SDImageCache用于做图片缓存，它是一个单例类。

缓存一开始注册监听了几个事件：

  • UIApplicationDidReceiveMemoryWarningNotification 内存警告通知
  • UIApplicationWillTerminateNotification 程序关闭通知
  • UIApplicationDidEnterBackgroundNotification 退到后台通知

内存警告通知用来清除NSCache中的缓存

关闭程序和退到后台的时候会清除掉时间过期的缓存，因为磁盘缓存默认只保存一周的时间，超过默认时间一周的文件将会被删除。

同时如果有设置过文件缓存大小限制，则按照时间删除老的缓存文件，直到所有文件数据大小为缓存限制的一半。

SDWebImageDownloader

SDWebImageDownloader用于配置下载选项和创建下载操作加入到下载队列中，它是一个单例类。

利用SDImageDownloder类进行下载时，其内部会创建一个SDWebImageDownloaderOperation下载操作，他是一个NSOperation的子类，然后将其添加到下载操作队列中NSOperationQueue中。
