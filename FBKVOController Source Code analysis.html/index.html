FBKVOController源码阅读

开发
  • Source Code
  • iOS


定义

键值观察(KVO)在苹果的官方文档中是这么定义的：它是一种允许观察者对象在所观察其他对象的属性发生改变时可以获得这些改变的通知这样一种机制。

但是原生的KVO在实现上存在着这么一些问题：

  • KVO在添加观察者后，在不再需要继续观察的时候应该将其移除。
  • 添加和移除需要成对出现，我们在开发的时候往往可能会由于疏忽导致代码里忘记在不需要使用的时候去释放掉观察者，那么这样就会埋下程序会crash的隐患。
  • KVO的所有获取改变通知有着统一的回调函数进行处理，如果观察者同时观察了多个属性的变化，那么这个回调可能会变得非常复杂。

FBKVOController是Facebook开源的一套基于KVO再封装的解决方案，相比较原生的KVO，它有如下这些优点：

  • 在注册观察者的时候提供了多种形式处理回调：block或者指定一个处理回调方法等等，从而达到了分散回调处理方法的目的。
  • 不再需要手动移除观察者了，FBKVOController会在内部自动在观察者controller释放掉的时候进行移除处理。
  • 在多线程环境下安全。

下面针对FBKVOController这个开源解决方案来继续的阅读其源码，看看其是如何实现这些改进的。

NSObject+FBKVOController

NSObject+FBKVOController.m是一个分类。通过关联对象给NSObject提供一个Retain和nonRetain的KVOController。

Retain和nonRetain的区别就是将被观察者添加到NSMapTable中是否将其引用计数+1。

_objectInfosMap是一个NSMapTable对象，它的作用和NSDictionary类似，但是又有区别：

NSDictionary提供了Key-to-Object的映射，并且Key必须是一个遵守NSCopying协议的不可变对象，如NSString，因为在添加到字典中的键值对，字典始终会对key进行一次拷贝，从而复制到其私有的位置。

NSMapTable是一个可变集合，没有对应的不可变版本，它提供的是一种一般意义上的映射，它与字典不同的是，它支持Object-to-Object的映射；在创建NSMapTable对象时会提供一些选项来决定Object的管理方式，无论是作为键还是值的Object都需要指明：

  1. NSMapTableStrongMemory (“memory option”)：添加后其引用计数+1
  2. NSMapTableWeakMemory (“memory option”)：不会修改引用计数，并且在对象释放后，会将其从容器中移除
  3. NSMapTableObjectPointerPersonality (“personality option”)：控制在将对象添加到集合中时是否调用对象上的isEqual:和hash方法。
  4. NSMapTableCopyIn (“copy option”)：添加的是复制的一份新的对象于集合中

这几行代码在我刚开始看的时候有点迷糊，捋顺之后其实不难理解：因为每一个所观察的属性都会被单独封装为一个_FBKVOInfo对象，而同时可能观察很多属性，所以要将这些单个的_FBKVOInfo对象放在集合中保存(NSMutableSet)，而NSMaptable以被观察者的引用为key，它默认使用的是强引用以防止被观察者释放，其对应的value就是所观察其属性NSMutableSet集合。

后面在不再继续需要观察某个属性的时候，就需要将其所观察的属性从集合中移除：因为FBKVOController是通关联对象绑定到观察者上的，所以在观察者释放的时候，关联的FBKVOController也会释放掉，在FBKVOController的dealloc方法中实现了自动解除观察的机制，无需再手动解除。

[7] 调用_FBKVOSharedController对象的observe:info:方法进行处理，其中_FBKVOSharedController是一个单例对象。

总结

到这里，FBKVOController源码的主要部分阅读就进行完毕了，总的来说该框架做了下面几件事情：

  • 提供懒加载的方式为对象关联一个FBKVOController来处理KVO观察，并且可以指定是否当被观察者添加到集合中对其引用计数+1。
  • 对每一个所观察的属性封装为_FBKVOInfo对象，对象中除了持有者所观察的属性keypath之外，还包括观察选项NSKeyBalueObservingOptions，回调方式，_FBKVOInfoState观察状态和FBKVOController对象的弱引用。
  • FBKVOController对象将所有的_FBKVOInfo对象集合添加到NSMapTable中进行管理，以观察者的强或者弱引用为键，以集合为值。
  • _FBKVOSharedController作为单例对象，集中处理所有的观察回调，并通过不同的方式传递给观察者。同时其内部维护着所有_FBKVOInfo对象弱引用集合，当对应的_FBKVOInfo对象释放后，会在集合中移除该对象，那么再观察回调中不再会对所观察的属性进行进一步处理。
  • 在FBKVOControllerdealloc时会移除掉所用的观察，无需再次手动释放，也就是将NSMapTable中的数据清空，并移除观察。
