配置私有CocoaPod库

开发
  • Tools
  • iOS

前言

目前在公司对项目在做组件化的工作，其过程势必需要对代码拆分，因为现阶段都是用CocoaPod对各种SDK的管理，故需要对拆分后的组件逐个进行私有Pod库的配置，进行管理，所以用这篇文章记录下整个配置过程以及中间遇到的问题。

流程

创建私有仓库

因为公司现在对代码都是在gitLab上管托，所以针对单个组件来说，就需要在gitLab上创建一个私有的仓库，管托单个组件的代码。

这里最好在创建仓库的时候，同时建立好LISENCE文件，因为之后的步骤需要用到。一般都是用的MIT LISENCE，其实只要找到其他一份LISENCE证书文件，复制里面的文字就可以了，然后对首行的文字进行修改，其他内容全部是一模一样，直接用就可以了，其格式如下:

Copyright (c) 2017 [Name] <Email>

顺便也可以同时创建README文件，虽然不是必要，但是在首页对当前仓库里的代码文件进行描述和说明也是很有用的。

组织目录

私有仓库创建好后，便可以使用git clone将远程仓库克隆到本地了，建议在clone的路径下面新建一个目录，用来存放拆分后的组件代码文件。

现在clone后下的目录有3样东西：

  • 新建的存放代码的文件夹

  • REDAME文件

  • LISENCE文件

坑

虽然整个过程步骤不是很多，但是我在做的时候还是碰到了1，2个比较头疼的问题。

pod linb lint 报错：source_files路径中找不到任何文件

这个.spec文件中的s.source_files字段是要将所有文件的路径写其的，有几个不同的路径就写多少，要不然就出现了这个错误。

pod repo push 报错 the repo at… is not clean

提交新的spec文件了这个错，后来发现原因是我直接在本地的spec仓库中对原来文件做了些操作，所以这里直接将改动的地方，提交到远程仓库中，然后再用pod repo push就可以了。

报错：没有权限提交代码到spec仓库中

这个我一开始直接感觉是SSH出问题了，但是检查发现SSH Key的配置没有错误，后来发现时.spec文件中s.source地址一定是http开头的路径，不能是git@开头的。

第三方框架或者静态库不支持i386与X86_64架构

一开始没找到解决方式，有种想哭的感觉，这个东西必须要用，但是又过不了验证，并且是第三方的东西，拿不到源码，无法让其支持模拟器……
