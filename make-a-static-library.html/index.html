静态库的制作过程

开发
  • iOS


起因

今天由于项目的需要，需要制作一份iOS静态库，所以在此记录下其制作过程。


流程

创建工程

首先打开Xcode新建一个工程，在Framework & Library栏下选择Cocoa Touch Static Library，然后点击Next，输入Product Name。


编写代码

创建.h文件和.m文件，并在其中编写所需要的代码；当然，也可以添加已经写好的文件。（这里写个简单的例子方便参考）

修改Scheme

在导航工具栏中，点击Edit Scheme

将Build Configuration中Debug修改为release。如果Destination选择iOS Device，那么制作的库只能在真机下运行，如果Destination选择模拟器，那么制作的库只能在模拟器下运行。


编译

关闭窗口，command+B重新编译程序，可以发现Products中的libmyFramework.a由红色变成黑色，右键点击库，选择Show in Finder就能找到.a静态库了，在Release-iphoneos文件夹下的.a库只能在真机上运行，而Release-iphonesimulator文件夹下的.a库只能在模拟器上运行，将其和头文件一起拷贝出来，就可以给其他工程用了，这就意味着一份静态库制作好了！

后记

这次制作库是为了考证，对于两份完全相同的库，调用其中的函数到底是调用哪个库的，经过测试找出了结论：在一个工程中引用了两份完全相同的库（存储在不同的文件夹下），调用其中的函数，是根据Library Search Paths的顺序来调用的，后面相同的库中的函数则不会再去调用。如下图所示，调用相同库中的函数，只会调用在路径$(PROJECT_DIR)/framework下的库中的函数，另一份库则不会去调用。
