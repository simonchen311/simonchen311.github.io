给App增加iTunes文件共享功能

从升级到IOS8.3后，发现使用iTools等第三方工具，已经不能使用其打开应用文档的功能进行文件导入/导出了，查询了网上，发现是IOS8.3版本以后已经关闭了对应用文件的访问权限。

可是手头的项目要测试，需要提取出应用中的文档，于是只能使用iTunes的文件共享功能来提取了，这需要进行一步简单的设置：

在Xcode中，在plist配置文件中增加“Application supports iTunes file sharing”，并将这一项的值设置为YES即可。

之后在真机上运行App，并在iTunes上查看，就可以共享文件给App或者导出App中的文件了。

全局修改BackBarButtonItem

iOS默认导航栏的返回按钮为箭头+文字显示，但是项目中要求不现实文字，只需要显示箭头，如果在父类控制器中进行修改需要按照如下的代码编写

但是这么做要求对所有的视图控制器进行修改，直接做么做会改到想哭……

vc的navigationItem.backBarButtonItem默认是nil的，而且只有在nil的时候，系统才会把vc的title当作后退文字来使用

解决方法是通过Method Swizzle轻松解决，实现在默认情况下就是只显示箭头而不带文字。如果有某个控制器自己对backBarButtonItem赋值的话，就按控制器自定义的来显示，这样我们才能总是得到真正想要的item。
