小技巧：EXC_BAD_ACESS的调试

开发
  • Skill
  • iOS

今天在项目运行时出现EXC_BAD_ACESS错误，导致程序崩溃，在开发时遇到EXC_BAD_ACESS这种错误不容易找到原因，但通常是由于内存管理问题而引起的，查询其解决方法，有如下几种办法可以来帮助查询出错的地方。

全局断点

进入断点面板，选择底部的“+”，单击Exception Breakpoint，无需修改任何设置即可实现全局断点的添加

设置好后，每当程序有异常抛出的时候，调试器就会中止程序运行并指向有问题的代码行。

NSZombieEnabled

在Xcode中的菜单栏中选择 Product -> Scheme -> Edit Scheme，打开窗口，设置MallocStackLogging和NSZombieEnabled两个环境变量查看错误来源。

最后记得把环境变量NSZombieEnabled，MallocStackLogging删除或设置为NO，因为它们会使得内存不会被释放。
